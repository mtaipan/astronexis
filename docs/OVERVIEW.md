# Обзор проекта Astronexis

## Что это

Astronexis — это **сервер Minecraft** (поддерживает Java- и Bedrock-игроков через
Geyser/Floodgate) вместе со всей обвязкой вокруг него:

- **Игровой плагин-ядро** (`AstronexisCore`) — внутриигровое меню, навигация (компас-хаб),
  action bar, мультиплатформенный рендер интерфейсов (сундук-меню для Java, формы для Bedrock).
- **Авторизация** (`TaipanAuthTg`) — поверх плагина AuthMe добавляет второй фактор:
  игрок при входе получает одноразовый код в Telegram и подтверждает его командой `/tgcode`.
- **Сайт** (`server-site`) — лендинг проекта, топ донатеров, страницы оферты/политики и
  приём платежей (донаты и VIP на 30 дней) через ЮKassa.
- **Telegram-бот** (`TgBot`) — точка входа для игроков вне игры: привязка ника к Telegram,
  проверка подписки на каналы, тикеты поддержки, админ-панель.

Связующий идентификатор между всеми частями — **ник игрока** и **Telegram chat_id**.

## Как модули связаны

```
                         ┌──────────────────────┐
                         │   Игрок (Java/Bedrock) │
                         └───────────┬───────────┘
                                     │ заходит на сервер
                                     ▼
   ┌─────────────────┐      ┌────────────────────┐      ┌──────────────────┐
   │  AstronexisCore  │      │    TaipanAuthTg     │◀────▶│      AuthMe       │
   │  (меню, компас)  │      │  (2FA через TG)     │      │  (логин/пароль)  │
   └─────────────────┘      └─────────┬──────────┘      └──────────────────┘
                                      │ читает привязку ник↔chat_id
                                      │ шлёт код в Telegram
                                      ▼
                            ┌────────────────────┐
                            │  Postgres (bindings) │◀──────┐
                            └─────────┬──────────┘        │ та же таблица привязок
                                      │                    │
                                      ▼                    │
   ┌─────────────────┐      ┌────────────────────┐        │
   │   Игрок в TG     │─────▶│       TgBot         │────────┘
   │                  │      │ (bind, тикеты,      │
   └─────────────────┘      │  подписки, админка) │
                            └────────────────────┘

   ┌─────────────────┐      ┌────────────────────┐      ┌──────────────────┐
   │  Пользователь   │─────▶│     server-site     │─────▶│     ЮKassa        │
   │   на сайте      │      │  (донаты / VIP)     │◀─────│   (webhook)      │
   └─────────────────┘      └─────────┬──────────┘      └──────────────────┘
                                      │
                                      ▼
                            ┌────────────────────┐
                            │ Postgres (payments) │
                            └────────────────────┘
```

### Ключевые связи

1. **TaipanAuthTg ↔ TgBot ↔ Postgres.** Бот (`/bind`) записывает связь `ник ↔ chat_id`
   в таблицу `bindings`. Плагин при входе игрока читает эту связь и шлёт код в Telegram.
   В `TgBot/config.yaml` путь к `bindings.yml` указывает прямо в каталог плагина — то есть
   исторически связь была через YAML-файл, сейчас обе стороны умеют Postgres.
2. **server-site ↔ ЮKassa ↔ Postgres.** Сайт создаёт платёж в ЮKassa, сохраняет его в
   таблице `payments`, ЮKassa присылает webhook о статусе. Топ донатеров на главной
   считается из успешных платежей.
3. **Фулфилмент платежей в игру пока не реализован** — после оплаты VIP/доната нет
   автоматической выдачи прав на сервере (см. [SECURITY.md](SECURITY.md) и `SITE-3` в [TODO.md](TODO.md)).

## Технологический стек

| Слой | Технологии |
|---|---|
| Серверные плагины | Java 21, Gradle, Paper/Spigot API 1.21.1, AuthMe API, Floodgate/Geyser |
| Веб | Java 17+/Spring Boot 3.5.11, Thymeleaf, Spring Data JPA, Flyway, Lombok |
| Бот | Python 3.12, python-telegram-bot 21.6, SQLAlchemy 2.0 (async, asyncpg), Alembic |
| Хранилища | PostgreSQL (бот, авторизация, сайт), YAML-файлы (legacy/fallback) |
| Платежи | ЮKassa (YooKassa API v3) |
| Деплой | Docker, docker-compose |

## Глоссарий

- **Привязка (binding)** — связь игрового ника с Telegram chat_id. Ключи уникальности:
  один TG ↔ один ник на платформу; один ник ↔ один TG в рамках платформы.
- **Платформа (platform)** — `java` или `bedrock`. Bedrock-ники в БД хранятся с префиксом `bed_`.
- **2FA-код** — 6-значный одноразовый код входа, TTL 120 секунд, генерируется `SecureRandom`.
- **Trust-сессия** — после успешного ввода кода игрок «доверен» на `trustTtlSeconds`
  (по умолчанию 6 часов) с привязкой к отпечатку (IP / подсеть).
- **Gate / Freeze** — режим «заморозки» неавторизованного игрока (нельзя двигаться,
  получать урон, выполнять команды).
- **Фулфилмент** — выдача купленного товара (VIP-права, отметка доната) на игровом сервере.
- **Action bar** — строка над хотбаром игрока (используется для рекламы адреса сервера).

## Текущее состояние (по состоянию на ревью)

- Архитектурно проект зрелый: модульное ядро, слоистый бот, абстракции хранилищ.
- **Не готов к продакшену с реальными деньгами**: подделываемый webhook, секреты в репо,
  нет фулфилмента, нет тестов. Подробности — в [SECURITY.md](SECURITY.md) и [TODO.md](TODO.md).
- Миграция меню в `AstronexisCore` завершена; старый `TaipanMenuPlugin` удалён (ARCH-2).
