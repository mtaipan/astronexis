# Документация Astronexis

Это оглавление. Документы разбиты так, чтобы по ним можно было пошагово чинить проект.

## Порядок чтения

1. [OVERVIEW.md](OVERVIEW.md) — что это за проект, зачем каждый модуль, глоссарий терминов.
2. [ARCHITECTURE.md](ARCHITECTURE.md) — как всё устроено и как данные ходят между модулями.
3. [STATUS.md](STATUS.md) — **честная оценка** текущего состояния: что проверено, что нет,
   почему это пока не «хороший» проект.
4. [ROADMAP.md](ROADMAP.md) — путь от «не сломано» → «хорошо» → «отлично» (3 фазы).
5. [SECURITY.md](SECURITY.md) — уязвимости и риски (читать до деплоя).
6. [DECISIONS.md](DECISIONS.md) — архитектурные решения (ADR): **почему** код устроен так.
7. [TODO.md](TODO.md) — **главный рабочий документ**: полный список задач с чек-боксами,
   разбитый по приоритетам. Чинить идём отсюда.
8. [SETUP.md](SETUP.md) — запуск и сборка каждого модуля.

## Как пользоваться этим набором

- Каждый модульный документ имеет раздел **«Слабые места / TODO»** со ссылками на
  пункты в [TODO.md](TODO.md). Идентификаторы задач сквозные (например, `SEC-1`, `SITE-3`).
- В [TODO.md](TODO.md) задачи отсортированы по приоритету: 🔴 критично → 🟠 важно → 🟡 улучшения.
- Каждая задача оформлена как чек-бокс `- [ ]`. Отмечай `- [x]` по мере выполнения.

## Условные обозначения

- 🔴 — критично, блокирует безопасный продакшен.
- 🟠 — важно, влияет на корректность/надёжность.
- 🟡 — улучшение качества, не срочно.
- 🟢 — сильная сторона (что уже сделано хорошо).
