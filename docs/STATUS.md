# Состояние проекта: честная оценка

Документ намеренно без приукрашивания. Нужен, чтобы не путать «починили очевидно
сломанное» с «проект хороший». Обновлять по мере появления тестов и реальных прогонов.

## Короткий вердикт

Работающие дыры прошлого ревью (обход 2FA, мёртвая админка, гонка выдачи VIP, JDBC в тике)
**закрыты кодом и — впервые в проекте — автотестами**: 20 тестов в трёх модулях, все зелёные.
Плагины теперь реально собираются (JDK 21 установлен), бот получил рабочий venv.

Проект всё ещё **не** production-ready: ни один сквозной путь («оплата → грант → выдача»,
«вход → код → доступ») не прогнан вживую против реальных БД/Telegram/сервера, CI нет,
ручные действия по секретам (SEC-2) не сделаны. Но статус сместился с «есть работающие
дыры» на «известные дыры закрыты и защищены тестами от рецидива».

## Что НЕ сделано и где иллюзия надёжности

1. **Ничего не запускалось вживую end-to-end.** Тесты — модульные, с моками. Сайт не
   поднимался против Postgres, webhook ЮKassa не вызывался, бот не подключался к Telegram,
   плагин не ставился на сервер. «Оплата → грант → RCON-выдача» ни разу не прогнана целиком.
2. **CI нет.** Тесты живут локально; никто не гоняет их на каждый коммит. Следующий
   регресс останется незамеченным до следующего ручного прогона.
3. **Покрытие точечное.** Закрыты именно найденные дыры. Webhook-верификация (SEC-1),
   PaymentService, FxService, правила привязок, Codes/Fingerprints — без тестов (TESTS).
4. **SEC-2 (ручное) не сделано:** bot-token не перевыпущен, пароли БД не сменены.
5. **Архитектурные долги на месте:** плагин читает чужую таблицу `bindings` сырым SQL
   (ARCH-4), `vip_grants` никто не консюмит кроме RCON, контракта между модулями нет (ARCH-1).
6. **Операционная зрелость по-прежнему низкая:** нет линтеров, форматтеров, мониторинга,
   алертов, бэкапов.

## Куда дальше

2. **CI** (GitHub Actions или аналог): `mvnw test` + `gradlew build` + `pytest` на каждый
   коммит — иначе новые тесты быстро протухнут.
3. Сквозной прогон: поднять Postgres + сайт локально, дёрнуть webhook руками, увидеть грант
   в БД; поднять бота с тестовым токеном, прокликать админку.
