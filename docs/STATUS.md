# Состояние проекта: честная оценка

Документ намеренно без приукрашивания. Нужен, чтобы не путать «починили очевидно
сломанное» с «проект хороший». Обновлять по мере появления тестов и реальных прогонов.

## Короткий вердикт

Сейчас это **крепкий, аккуратно прибранный pet-проект с правильной архитектурой и
закрытыми грубыми дырами**. Это **не** production-ready и пока **не** «хороший проект» в
профессиональном смысле. Мы перевели его из состояния «опасно» в «не стыдно показать» —
это снижение вреда, а не доказательство качества.

**Качество подтверждается не отсутствием видимых багов, а тестами и реальной работой.**
Ни того, ни другого пока нет.

## Что НЕ сделано и где иллюзия надёжности

1. **Ноль автоматических тестов.** Платёжный outbox, RCON-клиент, верификация webhook,
   правила привязок, гейт авторизации — ничего не покрыто. Для кода про деньги и доступ
   это дисквалифицирующий пробел.
2. **Ничего не запускалось вживую.** `BUILD SUCCESS` и `py_compile` означают «синтаксис
   верный», а не «работает». Сайт не поднимался против БД, webhook не вызывался, бот не
   стартовал (venv битый), сценарий «оплата → грант → выдача» не прогонялся ни разу.
3. **Плагины — review-only.** В окружении только JDK 26, Gradle 8.8 не стартует (OPS-4).
   Изменения в `GateListener`/`AstronexisCore` не компилировались.
4. **Наглядный урок про отсутствие тестов.** При рефакторинге бота декоратор по ошибке
   навесился на синхронный `is_admin` — это открыло бы админку всем. Поймано глазами;
   тест/запуск поймали бы это автоматически и гарантированно.
5. **Операционная зрелость ≈ 0.** Нет git/CI, линтеров, форматтера, мониторинга, алертов,
   бэкапов, стратегии логов. Интеграция «сайт → плагин» для грантов пока на словах
   (консьюмер не написан; БД у сайта и бота разные).

## Планка «хорошего» (минимум)

- [ ] Тесты на платёжную и привязочную логику (webhook-верификация, идемпотентность outbox,
      правила `bind_strict`, лимит кода, guard).
- [ ] Хотя бы **один реальный сквозной прогон** каждого пути, задокументированный.
- [ ] Сборка плагинов на JDK 21.
- [ ] git + CI, который гоняет сборку и тесты на каждый коммит.

Пока эти пункты не закрыты, статус остаётся «не хороший, но безопаснее».
